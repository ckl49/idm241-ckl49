Case Study

Overview

For this microinteractions course, I designed a Glossier-style button and expanded it into a full product card using HTML, CSS, and JavaScript. The goal was to create a clean, minimal interaction that reacted smoothly to user input. What began as a single button grew into a multi-state card with flavor selection, image changes, hover animations, an add-to-cart button, and a heart button.


Here's a link to their original website : Glossier

Problem

I needed to design an interaction that communicated clear feedback and stayed consistent with Glossier’s aesthetic. A static animation was not enough, so I incorporated JavaScript to handle logic, state management, and movement.

Goals

  • Build interactive behavior with JavaScript
  • Understand microinteraction rules like triggers, rules, feedback, loops and modes
  • Practice animating and timing in code
  • Manage UI states like flavor selection and image swapping
  • Strengthen DOM manipulation and debugging skills

Process

I studied Glossier’s UI patterns. Key traits included minimalism, soft transitions, simple color palettes, and subtle hover states. These guided the visual and interaction style.

I first sketched button states, then expanded the idea into a full product card. I also mapped out how each interaction would work and separated what CSS could handle from what required JavaScript. To aid in the development process, I utilized Figma to create prototypes, tinkering around with their new AI prototype builder, Figma Make. This was especially helpful in seeing clearly what I was going to replicate.

Development

To begin, I wrote in pure HTML and CSS to build out the static elements of the cards. I knew that I could only go so far with just CSS, so I turned to Javascript to aid in event trackers and state changes.

Learning JavaScript was the main challenge. Early issues included selecting elements, managing active states, and updating images reliably. I improved the structure by moving repeated logic into functions.

A turning point was using an associative array to match flavors to image paths. This kept the code clean and scalable, and ensured that all my data could live somewhere since I wasn't employing a backend or database. This would give functionality to my slideshow.

I smoothed transitions, fixed a double-click bug, refined spacing, and adjusted easing to make the interaction feel softer.

Final Features

  • Hover transitions
  • Click feedback
  • Flavor selection with image updates
  • Active state styling
  • A microinteraction aligned with a real brand aesthetic

From left to right: Slideshow, Show More Options, Liking a Product

Conclusion

The finished card was responsive and polished, and it became a strong example of how microinteractions support brand identity. It also strengthened my confidence in front-end development and JavaScript.

Key Learnings

  • JavaScript is essential for meaningful interaction
  • Associative arrays simplify state-based UI logic
  • Debugging is part of the design process
  • Good microinteractions blend design and engineering

Casey Lee, IDM 241
